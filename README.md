## Overview

## Project Purpose

The goal of this project was to recreate a web game such as Wordle, Minesweeper, or Solitaire using concepts thought throughout the course. The project emphasized collaborative software development, object-oriented design, and iterative development while working as part of a development team.

## What I Learned

This project significantly improved my understanding of object-oriented programming. It helped me gain practical experience and appreciate the importance designing systems where classes have clear responsibilities and interact through well-defined interfaces, using git collaboratively, balancing project goals with deadlines and conflicting schedules.

The hardest lesson and most valuable one came from working in a team of four; prior to this I had mostly worked alone and once with a partner. Good software engineering requires communication (be it through documentation, messaging, conversation) and compromise (there are many ways to do the same thing but maintainability and scalability are best achieved when everyone is on the same page, i.e. everyone agrees on coding standards, architecture, and development practices.).

As the project progressed, I had to implement solutions differently than I would have preferred in order to integrate with existing code and maintain compatibility with team members' work. This experience reinforced an important lesson: the "best" technical solution is not always the one that best serves the project, if it cannot be maintained consistently by the team.

Another memorable lesson came during our final demonstration rehearsal, when we discovered a bug involving repeat letters within the mystery word that had gone unnoticed during normal testing. This taught me how important comprehensive testing is; in this case Black Box testing as we only test the code, not the system.

lastly, the biggest lesson I learned was the importance of soft skills in engineering such as  coordinating schedules, resolving differing opinions, encouraging collaboration, ensuring progression, and sharing timely feedback. Overall, these experiences strengthened my appreciation for strong leadership, clear communication, consistent coding standards, and thorough testing. They also gave me a much better understanding of the interpersonal skills required to build software successfully as part of a team.
